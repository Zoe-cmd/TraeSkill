# Project Plan Template - 项目计划模板

## 使用说明

本模板用于 Project Manager 在 Phase 0 制定项目计划。项目计划是整个项目的纲领性文档，定义项目范围、里程碑、角色分配和风险。请完整填写所有章节。

## 4. 里程碑

| 里程碑 | 阶段 | 交付物 | 验收标准 |
|--------|------|--------|----------|
| M1: 项目规划完成 | Phase 0 | 项目计划、Todo 清单、决策日志 | Human Developer 审批通过 |
| M2: 需求与设计完成 | Phase 1 | PRD、UI 设计稿、架构文档 | PRD 评审通过 (G1) |
| M3: 后端开发完成 | Phase 2 | 数据库 Schema、API 文档 | API 评审通过 |
| M4: 前端与 AI 完成 | Phase 3 | 前端页面、AI 功能 | 前端可演示 |
| M5: 质量保证通过 | Phase 4 | 测试报告、安全审计报告 | 测试通过 (G4)、安全审计通过 (G5) |
| M6: 部署上线 | Phase 5 | 部署配置、CI/CD | 上线确认 (G6) |

## 7. 沟通计划

| 沟通项 | 频率 | 参与者 | 方式 | 产出 |
|--------|------|--------|------|------|
| 阶段交接 | 每阶段结束时 | 当前角色 → 下一角色 | 交接文档 | 交接文档存入 `docs/交接/` |
| 里程碑评审 | 每里程碑 | Human Developer + 相关角色 | 审批 | 门禁通过/阻塞记录 |
| 风险审查 | 每阶段 | Project Manager | 风险登记更新 | 更新本文档第 6 节 |
| 缺陷修复 | 发现 Critical/High Bug 时 | QA → 修复负责人 | 缺陷修复交接 | `docs/交接/缺陷修复交接-{BUG-ID}.md` |

## 8. 质量门禁

| 门禁 | 检查内容 | 通过标准 | 关联里程碑 |
|------|---------|---------|------------|
| G1 | PRD 评审 | Human Developer 审批通过 | M2 |
| G2 | 架构评审 | 技术负责人审批通过 | M2 |
| G3 | 代码审查 | 无 Critical/High 问题 | M5 |
| G4 | 测试通过 | 所有测试用例通过 | M5 |
| G5 | 安全审计 | 无 Critical 安全漏洞 | M5 |
| G6 | 上线确认 | Human Developer 审批通过 | M6 |
