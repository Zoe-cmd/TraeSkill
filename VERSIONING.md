# 版本管理协议 (VERSIONING)

> 本文件定义 TraeSkill 仓库自身的版本编号规则与迁移协议，确保仓库更新后旧项目能正确识别版本差异并执行迁移。

---

## 语义化版本 (Semantic Versioning)

TraeSkill 采用 `MAJOR.MINOR.PATCH` 三段式版本号：

| 版本段 | 触发条件 | 示例 | 迁移要求 |
|--------|----------|------|----------|
| **MAJOR** | 核心规则变更、阶段定义变更、工作流重大重构 | 1.0.0 → 2.0.0 | 需 Human Developer 确认，旧项目需手动迁移 |
| **MINOR** | 新增角色/阶段/规范文件/模板、功能增强 | 1.0.0 → 1.1.0 | 自动提示迁移，用户确认后继续 |
| **PATCH** | Bug 修复、文档校对、措辞优化 | 1.0.0 → 1.0.1 | 静默更新，无需迁移 |

## 迁移协议

### 项目版本检测

每个项目在 `docs/项目元数据.md` 中记录创建时使用的 TraeSkill 版本号。激活角色时，协调者按以下流程检测版本：

```
1. 读取 docs/项目元数据.md 中的 skill_version 字段
2. 与 SKILL.md 中声明的当前版本比较
3. 若版本不一致，按下表执行迁移
```

### 迁移矩阵

| 版本差异 | 迁移动作 | 用户交互 |
|----------|----------|----------|
| PATCH 差异 (1.0.0 → 1.0.1) | 无需迁移，静默继续 | 无 |
| MINOR 差异 (1.0.0 → 1.1.0) | 提示新增功能，更新项目元数据版本号 | 提示用户新版本变化，确认后继续 |
| MAJOR 差异 (1.0.0 → 2.0.0) | 暂停流程，展示迁移指南 | 必须用户确认后才能继续 |

### 项目元数据文件格式

```markdown
# 项目元数据

| 字段 | 值 |
|------|-----|
| 项目名称 | {项目名称} |
| 创建日期 | {YYYY-MM-DD} |
| TraeSkill 版本 | {版本号} |
| 当前阶段 | {Phase N} |

## 版本迁移记录

| 迁移日期 | 原版本 | 新版本 | 迁移类型 | 说明 |
|----------|--------|--------|----------|------|
| {YYYY-MM-DD} | 1.0.0 | 1.1.0 | MINOR | 自动迁移 |
```

## 变更分类指引

| 变更类型 | 版本段 | 判定标准 |
|----------|--------|----------|
| 新增角色文件 | MINOR | `roles/` 下新增 `.md` 文件 |
| 新增共享规范 | MINOR | `shared/` 下新增 `.md` 文件 |
| 新增模板 | MINOR | `templates/` 下新增 `.md` 文件 |
| 新增阶段 | MINOR | `workflow.md` 新增 Phase 定义 |
| 修改核心规则 | MAJOR | `SKILL.md` 核心规则条目变更 |
| 修改阶段定义 | MAJOR | 已有 Phase 的角色/产出物变更 |
| 修改工作流 | MAJOR | 角色激活/交接流程变更 |
| 修复文档矛盾 | PATCH | 计数错误、路径错误、命名不一致 |
| 补充说明 | PATCH | 新增注释、示例、注意事项 |
