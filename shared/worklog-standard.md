# Worklog Standard - 工作日志与防漂移规范

## 概述

本文档定义了 AI Agent 在单角色长对话中防止「上下文漂移（Context Drift）」的机制。当单个 Agent 在同一对话中连续处理多个子任务时，随着对话轮次增加，模型对初始角色设定、任务边界和已完成进度的保持能力会逐渐衰减，产生身份偏移、职责越界、进度遗忘等问题。本规范通过工作日志持久化、角色锚定声明和三级上下文预警三套机制，确保 Agent 在长对话中始终保持身份一致、边界清晰、进度可追溯，并在必要时通过新对话安全恢复。

## 适用范围

- 本规范适用于所有 AI Agent 单对话多子任务场景
- 当单个 Agent 在同一对话中预计处理 2 个及以上子任务时，必须启用本规范
- 多 Agent 协作场景中的交接与记忆管理另行遵循 `handoff-standard.md` 和 `memory-standard.md`，本规范聚焦单角色纵向防漂移

## 语言规范

本规范文档及所有相关文档、工作日志内容均使用简体中文编写。技术术语（如 Context Drift、Checkpoint、Snapshot、Anchor、Token、API、Git、CI/CD、Markdown 等）保留英文原文。日志文件中的代码标识符（文件路径、变量名、函数名等）使用英文。

## 防漂移原则

| 原则 | 说明 |
|------|------|
| 早锚定 | Anchor early，对话启动即声明身份，不等到漂移发生才纠正 |
| 勤记录 | Log often，每个检查点和停止时刻必须持久化进度 |
| 防漂移 | Stop before drift，达到预警阈值主动停止，不冒险继续 |
| 可恢复 | Recoverable，任何时刻停止后，新对话能从日志无损恢复 |
| 单一身份 | Single identity，整个任务链中角色身份与边界保持不变 |
| 最小增量 | 单次日志追加不超过 30 行，避免日志本身成为漂移源 |

## 工作日志

### 文件位置

工作日志统一存放在项目 `docs/` 目录下，按角色代号命名：

```
docs/工作日志-{角色代号}.md
```

### 写入时机

工作日志采用只追加（Append-only）模式，在以下三个时机写入：

| 时机 | 触发点 | 写入内容 | 目的 |
|------|--------|----------|------|
| 启动恢复 | 对话启动时 | 读取上次停止快照，输出恢复声明 | 重建上下文，确认起点 |
| 检查点 | 每完成一个子任务时 | 进度快照 + 已修改文件 + 锚定记录 | 固化阶段性成果 |
| 停止快照 | 对话结束或触发红色预警时 | 完整进度 + 未完成子任务 + 下次起点 | 支撑新对话无损恢复 |

> 启动恢复时机以「读取」为主，若上次无停止快照则创建首条日志；检查点和停止快照为「写入」时机。

### 日志格式模板

日志文件由头部声明和若干追加块组成。头部声明只写一次，后续每次写入均追加一个块。

#### 文件头部（仅创建时写入一次）

```markdown
# 工作日志 - {角色名称}（{角色代号}）

- 角色身份: {角色名称}
- 职责范围: {本角色负责的工作范围}
- 边界声明: {本角色不触碰的工作范围}
- 任务来源: {任务清单编号或 PRD 引用}
- 创建时间: {YYYY-MM-DD HH:MM}

---

## 日志记录
```

#### 单次追加块模板（检查点 / 停止快照通用）

```markdown
### {CHECKPOINT|SNAPSHOT}-{YYYYMMDD}-{NNN} | {时机} | {YYYY-MM-DD HH:MM}

- 角色身份: {角色名称}（{角色代号}）
- 当前子任务: {子任务编号} - {子任务标题}
- 状态: {COMPLETED|IN_PROGRESS|NOT_STARTED}
- 累计完成子任务数: {N}

进度快照:
- {本子任务已完成的具体事项 1}
- {本子任务已完成的具体事项 2}
- {本子任务已完成的具体事项 3}

已修改文件清单:
- {文件路径 1} - {新增|修改|删除} - {简述}
- {文件路径 2} - {修改} - {简述}

锚定记录:
- 身份: {角色名称}，未发生身份偏移
- 边界: {是否越界} - {越界说明或"无"}
- 下一子任务: {子任务编号} - {子任务标题}
- 备注: {红色预警停止点 / 阻塞事项 / 无}
```

### 追加行数限制

- 单次追加块不超过 30 行（含空行和分隔符）
- 若单次记录内容超过 30 行，必须拆分为多个块或精简表述
- 限制目的：避免日志本身占用过多 Token，成为新的上下文漂移源

## 角色锚定规则

### 锚定声明时机

每完成一个子任务，Agent 必须在输出中显式给出锚定声明，并在工作日志的锚定记录中登记。锚定声明是 Agent 对自身身份和边界的主动校验，不依赖外部提醒。

### 锚定声明模板

```markdown
【锚定声明】
- 身份: 我是{角色名称}（{角色代号}），职责为{职责范围}。
- 边界: 我的工作范围是{边界内事项}；{边界外事项}不属于我的职责。
- 越界自检:
  - [x] 未修改非本角色职责范围内的文件
  - [x] 未做出超出本角色权限的决策
  - [x] 未改变其他角色已确定的方案
  - [x] 未触发其他角色的工作流
- 结论: {锚定通过 | 检测到越界，已回退并纠正}
```

### 越界自检清单

锚定声明中的越界自检包含以下 4 项，任一项不通过即判定为越界，必须立即回退并纠正：

| 序号 | 自检项 | 说明 |
|------|--------|------|
| 1 | 是否修改了非本角色职责范围内的文件 | 如后端工程师修改了 UI 设计文件、数据库工程师修改了前端组件 |
| 2 | 是否做出了超出本角色权限的决策 | 如测试工程师擅自调整架构方案、前端工程师变更数据库 Schema |
| 3 | 是否改变了其他角色已确定的方案 | 如推翻架构师已 Approved 的技术选型、修改产品经理已定稿的需求 |
| 4 | 是否触发了其他角色的工作流 | 如未经交接直接启动测试流程、跳过 Code Review 直接部署 |

## 上下文预警机制

### 三级预警

| 级别 | 触发条件 | 级别含义 |
|------|----------|----------|
| 黄色预警 | 累计完成 3 个子任务 | 注意级，上下文开始累积，需加强锚定 |
| 橙色预警 | 累计完成 5 个子任务 | 警告级，漂移风险显著上升，需强制记录 |
| 红色预警 | 累计完成 7 个子任务 | 停止级，必须停止当前对话，转入新对话 |

### 各级别触发条件和动作

#### 黄色预警（3 个子任务）

- 触发条件: 本对话内累计完成子任务数达到 3
- 动作:
  1. 输出黄色预警声明
  2. 复核当前角色身份与边界（执行一次完整越界自检）
  3. 继续执行下一子任务，但提高锚定频率

#### 橙色预警（5 个子任务）

- 触发条件: 本对话内累计完成子任务数达到 5
- 动作:
  1. 输出橙色预警声明
  2. 强制写入检查点日志（无论当前子任务是否完成）
  3. 评估剩余子任务量，若超过 2 个则建议结束当前对话
  4. 若继续执行，每个子任务后必须写入检查点

#### 红色预警（7 个子任务）

- 触发条件: 本对话内累计完成子任务数达到 7
- 动作:
  1. 输出红色预警声明
  2. 立即停止执行新子任务
  3. 写入停止快照，记录未完成子任务和下次起点
  4. 提示用户在新对话中恢复，不得在当前对话继续

### 预警输出模板

```markdown
【{黄色|橙色|红色}预警】
- 已完成子任务数: {N}
- 触发阈值: {3|5|7}
- 当前角色: {角色名称}（{角色代号}）
- 当前子任务: {子任务编号} - {子任务标题}
- 建议动作: {复核锚定 | 强制检查点 + 评估收尾 | 停止并写入快照}
- 下一子任务: {子任务编号} - {子任务标题}
```

### 恢复流程

当对话因红色预警停止或主动结束后，新对话必须按以下流程恢复：

```
新对话启动
    ↓
读取工作日志 docs/工作日志-{角色代号}.md
    ↓
定位最近一条停止快照（SNAPSHOT 块）
    ↓
重新锚定（输出锚定声明，恢复身份与边界）
    ↓
确认未完成子任务列表和下次起点
    ↓
从最近一个未完成子任务继续执行
```

恢复时必须在日志中写入一条启动恢复记录，格式如下：

```markdown
### RECOVERY-{YYYYMMDD}-{NNN} | 启动恢复 | {YYYY-MM-DD HH:MM}

- 角色身份: {角色名称}（{角色代号}）
- 恢复来源: {上一条快照编号}
- 上次停止时间: {YYYY-MM-DD HH:MM}
- 已完成子任务数: {N}
- 恢复起点子任务: {子任务编号} - {子任务标题}
- 锚定状态: 已重新锚定，身份与边界已确认
```

## 反模式

| 反模式 | 说明 | 正确做法 |
|--------|------|----------|
| 无日志裸跑 | 在单对话中连续执行多个子任务却不记录工作日志 | 启动即创建日志文件，每个检查点追加记录 |
| 锚定缺失 | 完成子任务后不输出锚定声明，任由身份与边界漂移 | 每个子任务完成后强制输出锚定声明并完成 4 项越界自检 |
| 越界静默 | 越界自检发现问题却不回退纠正，继续推进 | 自检不通过立即回退，纠正后再继续并记录到日志 |
| 日志膨胀 | 单次追加超过 30 行，或在日志中堆砌无关信息 | 单次追加控制在 30 行内，只记进度、文件、锚定三类信息 |
| 忽视预警 | 达到橙色或红色预警阈值仍继续执行新子任务 | 橙色强制检查点，红色必须停止并写入快照 |

## 检查清单

- [ ] 工作日志文件路径符合 `docs/工作日志-{角色代号}.md` 规范
- [ ] 每完成一个子任务后输出了锚定声明并完成 4 项越界自检
- [ ] 单次日志追加块不超过 30 行，仅包含进度、文件、锚定三类信息
- [ ] 累计完成子任务数达到 3/5/7 时分别触发黄/橙/红色预警并执行对应动作
- [ ] 红色预警或对话结束时已写入停止快照，记录未完成子任务和下次起点
- [ ] 新对话启动时已读取工作日志、重新锚定并从最近未完成子任务继续
